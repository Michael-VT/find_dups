# find_dups: Многоязычный поиск дубликатов файлов

Комплексный поиск дубликатов файлов, реализованный на Go, Python и Rust с идентичными алгоритмами для сравнения производительности и производственного использования.

## Обзор

`find_dups` рекурсивно сканирует одну или несколько директорий, определяет дубликаты файлов с использованием хеширования SHA-256 и генерирует отчёты и скрипты удаления. Использует параллельную обработку для эффективной работы с большими коллекциями файлов.

### Основные возможности

- **Многоязычная реализация**: версии на Go, Python и Rust для сравнения производительности
- **Параллельное хеширование**: использует все ядра процессора для быстрого определения дубликатов
- **Безопасность**: генерирует скрипт удаления, а не удаляет файлы напрямую
- **Подробные отчёты**: экспорт в CSV с метаданными файлов и метками времени
- **Поддержка нескольких дисков**: может сканировать несколько директорий на разных точках монтирования

## Алгоритм

Все три реализации следуют одному и тому же алгоритму:

1. **Сбор файлов** — рекурсивный обход всех указанных директорий, запись пути, размера, времени создания и модификации
2. **Группировка по размеру** — только файлы с одинаковым размером (более одного файла) переходят к хешированию (оптимизация)
3. **Параллельное хеширование SHA-256** — вычисление криптографических хешей параллельно:
   - Go: горутины с пулом воркеров на основе каналов
   - Python: `multiprocessing.Pool`
   - Rust: параллельный итератор `rayon`
4. **Определение дубликатов** — группировка файлов по хешу внутри групп размера; все файлы в хеш-группе с >1 элементом являются дубликатами
5. **Генерация выходных данных**:
   - `duplicates_<lang>.csv` — все группы дубликатов с полной метадатой
   - `sort_dup_<lang>.csv` — все файлы, отсортированные по размеру (по убыванию)
   - `duprm_<lang>.sh` — bash-скрипт, удаляющий все дубликаты кроме первого (по ID) в каждой группе

## Выходные файлы

### duplicates_<lang>.csv
CSV-файл, содержащий все дубликаты файлов, сгруппированные по содержимому. Колонки:
- `FileID`: последовательный идентификатор файла
- `Path`: полный путь к файлу
- `Size`: размер файла в байтах
- `Hash`: хеш SHA-256 (шестнадцатеричный)
- `CreationTime`: время создания файла (ISO 8601)
- `ModificationTime`: время модификации файла (ISO 8601)

### sort_dup_<lang>.csv
CSV-файл, содержащий все просканированные файлы, отсортированные по размеру (по убыванию). Те же колонки, что и `duplicates_<lang>.csv`.

### duprm_<lang>.sh
Исполняемый bash-скрипт, который удаляет дубликаты файлов, сохраняя первый файл (наименьший FileID) в каждой группе дубликатов. **Просмотрите этот скрипт перед выполнением**, чтобы убедиться, что вы не удалите важные файлы.

## Установка и использование

### Реализация на Go

**Требования**: Go 1.16+

**Сборка**:
```bash
cd find_dups_go
go build -o find_dups_go find_dups_go.go
```

**Запуск**:
```bash
./find_dups_go /путь/к/сканированию1 /путь/к/сканированию2 ...
```

**Зависимости**: только стандартная библиотека

### Реализация на Python

**Требования**: Python 3.8+

**Запуск**:
```bash
cd find_dups_pthon
python3 find_dups_python.py /путь/к/сканированию1 /путь/к/сканированию2 ...
```

**Зависимости**: только стандартная библиотека

### Реализация на Rust

**Требования**: Rust 1.70+, Cargo

**Сборка**:
```bash
cd find_dups_rust
cargo build --release
```

**Запуск**:
```bash
./target/release/find_dups_rust /путь/к/сканированию1 /путь/к/сканированию2 ...
```

**Зависимости** (см. `Cargo.toml`):
- `walkdir` 2.5 — обход директорий
- `sha2` 0.10 — хеширование SHA-256
- `csv` 1.4 — запись CSV
- `chrono` 0.4 — форматирование времени
- `rayon` 1.12 — параллельная обработка

## Результаты бенчмарков

Протестировано приблизительно на 149 000 файлов в двух директориях (локальный SSD + внешний USB-диск):

| Метрика                | Python     | Rust       | Go         |
|------------------------|------------|------------|------------|
| Файлов просмотрено     | 149 044    | 148 819    | 148 819    |
| Файлов захешировано    | 128 964    | 128 738    | 128 738    |
| Время хеширования      | 1:55.751   | 2:07.520   | 1:31.992   |
| Общее время            | 5:26.243   | 3:55.664   | 3:06.040   |
| Дубликатов найдено     | 0          | 696        | 696        |
| Воркеров/потоков       | 12         | 12         | 12         |

**Примечания**:
- Время в формате `минуты:секунды.миллисекунды`
- Реализация на Python нашла 0 дубликатов, в то время как Go и Rust нашли 696, что указывает на возможную ошибку в логике определения дубликатов в версии Python
- Go показывает лучшую общую производительность, несмотря на более медленный сбор файлов, чем у Rust

## Оценка и рекомендации

### Достоинства

- **Практическая ценность**: высокая — решает реальную проблему поиска дубликатов в нескольких директориях и на разных дисках
- **Безопасность**: хорошая — генерирует скрипт удаления для просмотра, а не удаляет напрямую
- **Производительность**: все три реализации эффективно используют параллельную обработку
- **Прозрачность**: CSV-отчёты позволяют провести детальный анализ перед удалением

### Известные проблемы

1. **Расхождение в Python**: версия Python нашла 0 дубликатов, в то время как Go/Rust нашли 696. Это требует исследования — вероятно, связано с различным количеством файлов (149 044 против 148 819) или ошибкой в логике определения дубликатов.

2. **Ограничения платформы**:
   - Go использует `syscall.Stat_t` (специфично для macOS) для времени создания
   - Rust использует `std::os::darwin::fs::MetadataExt` для времени создания
   - Обе требуют условной компиляции для поддержки Linux/Windows

### Какую реализацию использовать?

- **Для производственного использования на macOS**: Go — самый быстрый в целом, один бинарный файл без зависимостей
- **Для кроссплатформенной разработки**: Rust — проще всего адаптировать с атрибутами `#[cfg(target_os)]`
- **Для быстрого написания скриптов/прототипов**: Python — проще всего модифицировать, но сначала нужно исследовать ошибку определения дубликатов

## Перспективы развития

1. **Исправить определение дубликатов в Python** — исследовать расхождение между реализациями
2. **Индикатор прогресса** — добавить отображение прогресса в реальном времени во время хеширования
3. **Кроссплатформенное время создания** — использовать условную компиляцию для Linux/Windows
4. **Оптимизация частичного хеширования** — хешировать первые/последние N КБ + размер перед полным хешем файла
5. **Настраиваемый вывод** — возможность указывать выходную директорию и префиксы файлов
6. **Интерактивный режим** — простой TUI для просмотра дубликатов перед удалением
7. **Режим dry-run** — показать, что будет удалено, без генерации скрипта
8. **Перемещение вместо удаления** — возможность перемещать дубликаты в staging-директорию
9. **Фильтр минимального размера** — пропускать файлы ниже настраиваемого порога (например, <1КБ)
10. **Дедупликация символическими/жёсткими ссылками** — замена дубликатов жёсткими ссылками для экономии места без удаления
